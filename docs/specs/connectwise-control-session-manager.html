Session Manager API reference CreateSessionAsync(sessionType, name, host, isPublic, code)

Deprecated for Support and Meeting sessions, use the updated form of CreateSession detailed next

Used to create a Support or Meeting session; parameters such as the assigned host, name, and method of joining are assigned at creation

Parameters

sessionType (SessionType)

Indicates the type of session to create (Note: only SessionType enumeration members Support and Meeting are supported for this method passing in Access will create an exception)

name (string)

Display name for the session; this is also the listed name visible on the Host page and in the public sessions dropdown menu of the Guest page

host (string)

The name of the session creator (Note: this does not cause the user to automatically join the session)

isPublic (bool)

Determines if the session is publicly available for joining and listed inside the dropdown menu on the Guest page

code (string)

The code that a Guest must enter on the Guest page to join the session

Return

Generates a new Session object with the specified parameters which then appears on the Host page of the web server (see Session in the Objects section for details)

CreateSessionAsync(byHost, sessionType, name, host, isPublic, code, customPropertyValues)

Used to create a Support or Meeting session; parameters such as the assigned host, name, method of joining and custom properties are assigned at creation

Parameters

byHost (string)

(Unused)

sessionType (SessionType)

Type of session to create (Note: only SessionType enumeration members Support and Meeting are supported for this method passing in Access will create an exception)

name (string)

Display name for the session; this is also the listed name visible on the Host page and in the public sessions dropdown menu of the Guest page

host (string)

The name of the session creator (Note: this does not cause the user to automatically join the session)

isPublic (bool)

Determines if the session is publicly available for joining and listed inside the dropdown menu on the Guest page

code (string)

The code that a Guest must enter on the Guest page to join the session

customPropertyValues (string[])

Any specified values for the eight possible custom properties

Return

Generates a new Session object with the specified parameters which then appears on the Host page of the web server (see Session in the Objects section for details)

Example

Automatically assign properties based on input arguments. In this case, input is used to determine the necessity of a session code. More importantly, the session is automatically assigned customPropertyValues based upon the session name. The example could be paired with specific Session Groups Filters and Permission Roles to automatically organize sessions and restrict access.

1public async Task CreateSupportSessionTest(string name, bool isPublic, string code)
2{
3code = isPublic ? null : code;
4string[] customPropVals = name.Contains("Test") ? new string[]{"Test"} : new string[0];
5await SessionManagerPool.Demux.CreateSessionAsync(null, SessionType.Support, name, null, isPublic, code,customPropVals);
6}

Warning: Supported extensions use a method named "CreateSupportSession." If you use this method name in an extension, it will conflict with supported extensions.
