These are the frequently asked questions about the Asio API.

What integration options are there?

There are currently a few options for Integrations to access Asio. First is the REST APIs that can be used in conjunction with the Integration builder accessible in your test environment. Second is direct Snowflake access, This will let you access your data en mass and pull for reporting in your tools. Last, there are legacy APIs, but new integrations should not be built to these.

Snowflake VS REST APIs?

Snowflake is used for general reporting. If you need large amounts of data to put in your dashboards, then Snowflake will help you gather that all in one place. The REST APIs are more commonly used to take action on records. You can pull smaller amounts of records, but in real time, to then update directly with the API.

For more information on Snowflake, please see the documentation here.

We advise you to consult Snowflake documentation or Snowflake Support for troubleshooting Snowflake accounts. If you have issues setting up the connector or getting access to your Asio data, contact ConnectWise Support, and we can help you integrate your data with your Snowflake account.

How do I get a test environment?

Please reach out to your account manager or email accountmanager@connectwise.com

Why are there no computers shown on my test environment's Devices screen?

Your test environment comes with lots of configurations to look like a real environment. The Devices will need to be linked to real devices to show and let you run scripts on them. To do this, you will need to follow this guide on how to Install the RMM Agent.

How do I authenticate to the Asio REST APIs?

The standard method of authentication is with a clientid and secret.

How do you create a note on an existing ticket?

To create a note on an existing ticket, you will send a POST request to the following endpoint:

POST /api/platform/v1/service/ticketing/tickets/{id}/notes

Here, {id} in the URL path should be replaced with the actual ID of the ticket you wish to add the note to. The ticket ID should not be part of the request body.

Your request body will look like this for a basic note:

view sourceprint?
1{
2  "detail": "this is the note that you want to add",
3  "visibility": 1
4}

For visibility, you will send either a 1 or a 2.

  • 1 makes the note public-facing
  • 2 makes it an internal note
What is the purpose of the extendedAttributes array in the request body?

The extendedAttributes array refers to custom data fields that you can optionally add to the note you are creating. Think of them as additional, specialized pieces of information you might want to associate with the note, beyond just the basic text.

Here's an example of how extendedAttributes is structured when used:

view sourceprint?
1"extendedAttributes": [
2 {
3  "id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
4  "name": "string",
5  "value": "string"
6  }
7]

Each object within the extendedAttributes array represents a single custom data field, with its own id, name, and value.

What if I just want to create a simple note without any custom data fields?

If you do not want to add any custom data fields to your note, you would leave the extendedAttributes array blank (empty) in your request body.

Here's the correct format for creating a simple note, including the extendedAttributes  array correctly:

{
  "detail": "string",
  "extendedAttributes": [],
  "visibility": 1
 }

Replace "string" with your desired note content and set the visibility. For visibility, you will send either a 1 or a 2.

  • 1 makes the note public-facing
  • 2 makes it an internal note
Why did I receive an error when trying to create a note with extendedAttributes?

The error you encountered was likely due to incorrectly referencing extendedAttributes. If you were trying to add your note content within an extendedAttributes structure, like providing an id and name for a custom field that doesn't exist, the system would throw an error. This happens because you were essentially trying to populate a custom data field that wasn't set up in the system, rather than simply providing the main note detail.

How do I provide the x-request-id?
  1. On the Product Creating page, right-click and select Inspect.
  2.  Now, under the Inspect tab, select Network.
  3. In the Headers section, you will find the x-request-id.
