Important API Updates

API Updates are emailed when Staging environments are updated and for any major changes and issues. To get the most current information on the APIs fill out the form below.

Release Notes

Reason for Request:


New Pagination Method

We are introducing a new paging method that will compliment the current paging.  This new paging is going to be a forward only page method.  You supply an index number and the number of results you want to get next.  The response will include a new index number for you to continue your paging until you run out of possible values.

Splitting Service and Project Tickets

We are splitting the Service and Project Tickets and are removing the Project relating fields from the Service/Tickets endpoint.  Project related calls will be under the project module starting with 2018.5.

CW-APP-ID Changes

We are working on a new Application registration process where you can register your integrations.  We will give ever integration a unique tracking id that allows us to view how many calls are being made to each endpoint and to analyze performance as well as figure out where any bottle necks are being created.

Conversion to REST *Complete

We are currently working on a new suite of REST APIs.  We are converting all of the current APIs over to REST and adding any enhancements that have been requested and fit.  The new APIs will grant access to all setup table objects required for their specific objects.

Please note that release schedules are subject to change
